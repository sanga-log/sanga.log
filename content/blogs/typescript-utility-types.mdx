## 왜 유틸리티 타입인가?

TypeScript를 쓰다 보면 기존 타입을 변형해서 새로운 타입을 만들어야 할 때가 많습니다. 직접 새 타입을 선언하는 대신 내장 유틸리티 타입을 활용하면 코드가 DRY해지고 타입 안전성도 올라갑니다.

## Partial / Required

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

// 모든 필드를 선택적으로
type UpdateUserDto = Partial<User>;
// → { id?: number; name?: string; email?: string; }

// 모든 필드를 필수로
type StrictUser = Required<Partial<User>>;
```

## Pick / Omit

```typescript
// 특정 필드만 선택
type UserPreview = Pick<User, 'id' | 'name'>;
// → { id: number; name: string; }

// 특정 필드를 제외
type UserWithoutId = Omit<User, 'id'>;
// → { name: string; email: string; }
```

## ReturnType / Parameters

```typescript
async function fetchUser(id: number): Promise<User> {
  return await api.get(`/users/${id}`);
}

// 함수의 반환 타입 추출
type FetchUserReturn = Awaited<ReturnType<typeof fetchUser>>;
// → User

// 함수의 매개변수 타입 추출
type FetchUserParams = Parameters<typeof fetchUser>;
// → [id: number]
```

## Record

```typescript
// 특정 키-값 구조를 가진 객체 타입
type RolePermissions = Record<'admin' | 'user' | 'guest', string[]>;
const permissions: RolePermissions = {
  admin: ['read', 'write', 'delete'],
  user: ['read', 'write'],
  guest: ['read'],
};
```

## Discriminated Union과 Extract / Exclude

```typescript
type Shape =
  | { kind: 'circle'; radius: number }
  | { kind: 'rect'; width: number; height: number };

// 특정 유니언 타입만 추출
type CircleOnly = Extract<Shape, { kind: 'circle' }>;
// → { kind: 'circle'; radius: number }
```

## 마치며

유틸리티 타입을 잘 활용하면 타입 선언의 중복을 줄이고 유지보수성이 크게 향상됩니다. 처음부터 외우려 하지 말고, 필요할 때마다 찾아서 쓰다 보면 자연스럽게 익숙해집니다.
