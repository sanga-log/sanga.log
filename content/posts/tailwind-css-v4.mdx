## Tailwind v4의 핵심 변화

가장 큰 변화는 **설정 파일이 사라졌다**는 점입니다. `tailwind.config.js` 대신 CSS 파일 내에서 직접 테마를 설정합니다.

## CSS-first 설정

```css
/* globals.css */
@import "tailwindcss";

@theme {
  --color-primary: oklch(70% 0.2 240);
  --font-sans: 'Noto Sans KR', sans-serif;
  --spacing-18: 4.5rem;
}
```

## 빌드 성능 향상

v4는 Rust 기반의 새 엔진인 **Oxide**를 사용합니다. 기존 대비 빌드 속도가 최대 10배 빠릅니다.

## 새로운 유틸리티들

```html
<!-- 컨테이너 쿼리 (플러그인 없이) -->
<div class="@container">
  <div class="@md:flex-row flex-col">...</div>
</div>

<!-- 3D 트랜스폼 -->
<div class="rotate-x-12 rotate-y-6 perspective-1000">...</div>

<!-- 동적 유틸리티 -->
<div class="grid-cols-[repeat(auto-fill,minmax(200px,1fr))]">...</div>
```

## 마이그레이션 방법

```bash
npx @tailwindcss/upgrade
```

자동 마이그레이션 툴이 대부분을 처리해줍니다. `@apply`나 커스텀 플러그인을 많이 쓴 프로젝트는 수동 검토가 필요합니다.

## 마치며

v4는 단순한 업그레이드가 아닌 패러다임의 전환입니다. 초기 설정 비용이 있지만 장기적으로 유지보수 편의성과 성능 모두 크게 향상됩니다.
