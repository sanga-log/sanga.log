## 문제 상황

신규 랜딩 페이지의 LCP가 4.2초로 측정되었습니다. Google의 권장 기준(2.5초 이하)을 크게 초과했고, SEO 점수에도 직접적인 영향을 주고 있었습니다.

## 측정 도구

- Chrome DevTools Lighthouse
- PageSpeed Insights
- Web Vitals Chrome Extension

## 원인 분석

```
LCP 요소: 히어로 이미지 (1920x1080, 2.3MB PNG)
TTFB: 1.1s
렌더 블로킹 리소스: 3개 (폰트 2개, CSS 1개)
```

## 개선 1: 이미지 최적화

```tsx
// ❌ 기존
<img src="/hero.png" alt="히어로" />

// ✅ 개선: Next.js Image 컴포넌트
import Image from 'next/image';

<Image
  src="/hero.webp"          // WebP 변환 (PNG 대비 65% 감소)
  alt="히어로"
  width={1920}
  height={1080}
  priority                   // LCP 요소에 priority 설정
  placeholder="blur"
  blurDataURL={blurDataUrl}
/>
```

## 개선 2: 폰트 로딩 전략

```html
<!-- preconnect로 폰트 서버 연결 미리 열기 -->
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />

<!-- font-display: swap으로 FOUT 방지 -->
<style>
  @font-face {
    font-display: swap;
  }
</style>
```

## 개선 3: Critical CSS 인라인화

Above-the-fold 렌더링에 필요한 CSS를 `<head>`에 인라인으로 삽입하여 렌더 블로킹을 제거했습니다.

## 결과

| 지표 | 개선 전 | 개선 후 |
|------|---------|---------|
| LCP  | 4.2s    | 1.2s    |
| FID  | 180ms   | 45ms    |
| CLS  | 0.18    | 0.02    |

Lighthouse 점수: 52 → 94

## 마치며

성능 최적화는 "무엇이 느린지"를 정확히 측정하는 것에서 시작합니다. 감각이 아닌 데이터를 기반으로 개선하면 효율적인 성과를 낼 수 있습니다.
